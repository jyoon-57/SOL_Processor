# 프로젝트 개요: SOL (Zero-Command 바이오필릭 조명 시스템)

## 1. 프로젝트 핵심 철학

- **바이오필릭 디자인 (Biophilic Design):** 태양의 일주기 리듬(Circadian Rhythm)과 자연광의 특성을 실내 조명으로 모사하여, 시간에 따라 색온도(CCT)와 밝기(Brightness)를 자연스럽게 조절해 사용자의 생체 리듬(멜라토닌/코르티솔 분비) 안정을 돕는다.
- **Zero-Command (무명령 패러다임):** 사용자가 앱이나 음성으로 조명을 제어하는 '인지적 마찰'을 없앤다. 시스템이 사용자의 행동 맥락(Context)을 알아서 파악하여 은연중에(Implicit) 조명 환경을 최적화한다.

## 2. 시스템 아키텍처 (3-Node 분산 구조)

본 프로젝트는 연산 부하와 발열을 관리하기 위해 3개의 물리적 노드로 분리되어 있다.

- **Node B (SOL Lens - 라즈베리파이):** 초소형 엣지 관찰자 겸 중계기 (Eye & Router). 카메라로 공간의 영상을 캡처하여 H.264 소프트웨어 인코딩 후 UDP 스트림으로 Node A에 송출한다. 동시에 BH1750 I2C 센서로 주변 조도를 측정해 Node A로 전달한다. Node A로부터 최종 조명 명령을 수신하면, pywizlight 라이브러리를 통해 로컬 망에 있는 Node C(WiZ 전구)로 비동기 API 명령을 중계(Relay)한다.
- **Node A (SOL Processor - 고성능 노트북):** **[현재 우리가 개발할 타겟]** 시스템의 메인 두뇌. Node B로부터 수신한 초저지연 영상 스트림을 분석해 사용자의 행위를 파악하고, 일주기 알고리즘을 융합하여 최적의 색온도(CCT) 및 조도(Brightness)를 도출한 뒤, 이 제어 명령을 다시 Node B로 전송한다.
- **Node C (SOL Light - WiZ 스마트 전구):** 조명 실행자 (Actuator). 상용 스마트 전구로서, 클라우드 서버를 거치지 않고 로컬 네트워크에서 직접 UDP 패킷을 수신하여 픽셀 단위의 색온도와 밝기를 즉각적으로 물리적 공간에 출력한다.

## 3. SOL Processor의 역할과 핵심 로직 (현재 개발 목표)

SOL Processor(`sol_processor.py`)는 다음의 파이프라인을 무한 반복해야 한다.

1. **Input (입력):** 카메라 영상 프레임과 조도 센서 데이터(현재는 가짜 데이터 `mock_lux` 활용)를 실시간으로 받아온다.

2. \*\*Process 1 (공간 인지 - 2-Step 하이브리드 분석):

- Step A. 상시 감시 (MediaPipe): 프레임마다 사용자의 33개 관절 좌표를 추출한다. 이전 좌표와 비교하여 특정 임계값(Threshold, 미세한 노이즈/Jitter 제외) 이상의 유의미한 움직임이 있는지 판단한다.
- Step B. 심층 분석 (MobileNet-3D): 유의미한 움직임이 감지되었을 때만(ANALYZING 상태) 프레임을 샘플링(건너뛰기)하여 16장 단위의 슬라이딩 버퍼(플립북)를 생성한다. 버퍼가 꽉 차면 OpenCV DNN을 통해 'reading', 'sleeping', 'sitting', 'watching TV', '부재중' 등 구체적인 행위를 추론한다.
  상태 유지: 큰 움직임이 없을 경우(STABLE 상태), 불필요한 AI 연산을 멈추고 마지막으로 확인된 행동(LAST_KNOWN_ACTION)을 계속 유지한다.

3. **Process 2 (일주기 연산):** 현재 시각을 기반으로 생물학적으로 적합한 기본 색온도(CCT)와 밝기(BRI)를 수학적으로 계산한다.

4. **Process 3 (맥락 융합):** Process 1에서 도출된 구체적인 행동 맥락과 Process 2의 일주기 데이터를 융합한다. (예: 'reading' 상태면 일주기 리듬보다 집중을 위한 백색광 우선, 'sleeping' 상태면 기준 밝기를 대폭 낮추고 색온도를 웜톤으로 보정)

5. **Output (출력/송신):** 계산된 최종 제어명령(예: CCT:3000,BRI:50)을 문자열 패킷으로 포장하여 Node B(Lens)의 수신 포트로 발송하고, AI가 뼈대를 추론한 시각적 결과와 현재 상태 텍스트를 노트북 화면(OpenCV GUI)에 실시간으로 띄워 모니터링한다.

## 4. 현재 개발 페이즈 (Phase 1: 하이브리드 프로토타이핑)

- 물리적 분리 보완: 라즈베리파이(Node B)와 조명(Node C) 하드웨어가 아직 물리적으로 통합되지 않은 상태이므로, Processor(Node A)의 코어 로직 선행 개발에 100% 집중한다.

- 영상 소스 대체: 원격 UDP 영상 스트림 수신 대신 노트북 내장 웹캠(cv2.VideoCapture(0))을 영상 소스로 사용한다.

- 통신 방어 로직: 명령 송신 소켓 통신 기능은 작성하되, 실제로 통신 에러가 나지 않도록 try-except로 방어적으로 구현한다.

- [중요] AI 도입 전략: 복잡한 TensorFlow 환경 설정 없이 즉각적인 기획 검증이 가능한 OpenCV DNN + MobileNet-3D (.onnx) 모델을 사용하여, 행동에 따른 조명 제어 전체 파이프라인이 매끄럽게 돌아가는지 확인한다.
